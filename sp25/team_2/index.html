SpyberryPi: Motion-Activated Video Surveillance System

Introduction

Our project team set out to create a motion-activated surveillance system using a Raspberry Pi, a webcam, and a motion sensor. The goal was to configure the system so it would only record video when motion was detected, storing the footage locally. We followed an Agile-inspired approach, organizing our efforts into sprints to focus on iterative progress and testing.

Sprint 1: Acquiring Hardware

  1. Went to NPS RoboDojo to explore hardware options
  2. Practiced with the Raspberry Pi in their lab
  3. Acquired our own Raspberry Pi
  4. Purchased a compatible webcam, motion sensor, and SD card via Amazon and the NEX

Background: We started our project by visiting the NPS RoboDojo, which provided an opportunity to explore different Raspberry Pi configurations and get familiar with the hardware. While the RoboDojo allowed us to practice with their Raspberry Pis, they didn’t have the additional peripherals we needed. We decided to purchase the remaining components — a compatible webcam, motion sensor, and SD card — from Amazon and the Naval Exchange (NEX) so we could continue development independently.

Sprint 2: Software Setup and Initial Testing

  1. Downloaded Raspberry Pi OS from the official website
  2. Flashed the OS onto the SD card
  3. Installed MotionEye surveillance software
  4. Connected and tested the webcam with Raspberry Pi
  5. Wired and tested the motion sensor

Background: We downloaded the latest Raspberry Pi OS and installed it onto our SD card. We then installed the MotionEye package, an open-source surveillance software that enables video streaming and motion-triggered recording. This phase involved a lot of troubleshooting: from getting the webcam to be recognized by the system, to calibrating the motion sensor’s sensitivity.

Sprint 3: Integration and File Storage

  1. Configured MotionEye to trigger recording on motion detection
  2. Set recording duration to 10 seconds per trigger
  3. Tested file saving paths to local SD card
  4. Attempted to save to USB drive (unsuccessful)

Background: We set up the system to record 10-second clips when motion was detected, saving them to the SD card. We tried to redirect the storage to a USB drive, but due to mounting issues and compatibility concerns, that didn't work. Still, we had a functional motion-activated recording system by the end of the sprint.
